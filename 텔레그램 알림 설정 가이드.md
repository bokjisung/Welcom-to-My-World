STEP 1. 텔레그램 봇 생성하기
──────────────────────────────────────────────────────────────

1-1. 스마트폰 또는 PC에서 텔레그램 앱을 열고
     검색창에 "@BotFather" 를 검색합니다.

1-2. BotFather 채팅창에 들어가서 아래 명령어를 입력합니다.
     (파란색 링크로 보이면 탭/클릭해도 됩니다)

     /newbot

1-3. 봇 이름을 입력하라고 합니다. 원하는 이름을 입력하세요.
     예) HJS Dashboard Bot

1-4. 봇 사용자명(username)을 입력하라고 합니다.
     반드시 영문으로, 반드시 "bot"으로 끝나야 합니다.
     예) hjs_dashboard_bot

1-5. 성공하면 아래와 같은 메시지가 옵니다.

     "Done! Congratulations on your new bot."
     "Use this token to access the HTTP API:"
     123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ

     이 토큰(숫자:영문 형태)을 복사해서 메모해 두세요.
     이것이 TELEGRAM_BOT_TOKEN 입니다.


──────────────────────────────────────────────────────────────
STEP 2. 내 CHAT_ID 확인하기
──────────────────────────────────────────────────────────────

2-1. 텔레그램 검색창에서 "@userinfobot" 을 검색합니다.

2-2. userinfobot 채팅창에 들어가서 아래를 입력합니다.

     /start

2-3. 봇이 아래와 같은 정보를 돌려줍니다.

     Id: 987654321
     First: 홍길동
     ...

     "Id:" 뒤의 숫자를 복사해서 메모해 두세요.
     이것이 TELEGRAM_CHAT_ID 입니다.

     ※ 주의: 새로 만든 봇에게 먼저 /start 를 보내야
       나중에 알림을 정상적으로 수신할 수 있습니다.
       STEP 1에서 만든 봇 (@hjs_dashboard_bot 등) 채팅창에 들어가서
       /start 를 한 번 눌러 두세요.

STEP 3. .env 파일에 토큰 및 ID 등록하기
──────────────────────────────────────────────────────────────

3-1. 프로젝트 루트의 .env 파일을 엽니다.
     (.env.example 을 복사해서 .env 로 이름을 바꾼 파일)

3-2. 아래 두 줄을 찾아서 실제 값으로 채워 넣습니다.

     TELEGRAM_BOT_TOKEN=123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ

     ※ 등호(=) 앞뒤에 공백 없이 입력해야 합니다.
     ※ 따옴표도 필요 없습니다. 숫자/문자 그대로 입력하세요.

3-3. 저장합니다.

     포함되어 있지 않으면 GitHub에 토큰이 그대로 올라갑니다.

     .gitignore 파일 열어서 아래 줄이 있는지 확인:
     .env

STEP 4. npm 패키지 설치 확인하기
──────────────────────────────────────────────────────────────

4-1. backend 폴더로 이동합니다.

     cd backend

4-2. package.json 파일을 열어서 "node-fetch" 가 있는지 확인합니다.

STEP 5. config/telegram.js 파일 확인 및 작성
──────────────────────────────────────────────────────────────

5-1. backend/config/telegram.js 파일을 엽니다.

5-2. 아래 형태의 코드가 있는지 확인합니다.
     없다면 아래 내용을 그대로 작성합니다.

─────────────────── telegram.js 내용 ───────────────────────

const fetch = require('node-fetch');

const BOT_TOKEN = process.env.TELEGRAM_BOT_TOKEN;
const CHAT_ID   = process.env.TELEGRAM_CHAT_ID;

/**
 * 텔레그램으로 메시지를 전송하는 함수
 * @param {string} message - 보낼 메시지 내용
 */
async function sendTelegram(message) {
  if (!BOT_TOKEN || !CHAT_ID) {
    console.warn('[Telegram] BOT_TOKEN 또는 CHAT_ID가 설정되지 않았습니다.');
    return;
  }

  const url = `https://api.telegram.org/bot${BOT_TOKEN}/sendMessage`;

  try {
    const res = await fetch(url, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        chat_id: CHAT_ID,
        text: message,
      }),
    });

    const data = await res.json();

    if (!data.ok) {
      console.error('[Telegram] 전송 실패:', data.description);
    }
  } catch (err) {
    console.error('[Telegram] 오류 발생:', err.message);
  }
}

module.exports = sendTelegram;

STEP 6. 알림을 보낼 라우터에서 호출하기
──────────────────────────────────────────────────────────────

6-1. 알림을 보내고 싶은 라우터 파일 상단에 아래 줄을 추가합니다.
     예) backend/routes/auth.js 상단

     const sendTelegram = require('../config/telegram');

6-2. 알림을 보내고 싶은 시점에 아래처럼 호출합니다.

     예시 A — 로그인 성공 시:
     await sendTelegram(`[로그인 성공] ${username} / IP: ${req.ip}`);

     예시 B — 새 방문자 세션 시작 시:
     await sendTelegram(`[방문자 접속] IP: ${ip} / UA: ${userAgent}`);

     예시 C — 로그인 실패 횟수 초과 시:
     await sendTelegram(`[경고] 로그인 실패 5회 초과 / IP: ${req.ip}`);

6-3. await 를 빠뜨리지 않도록 주의합니다.
     해당 함수가 있는 곳의 바깥 함수에 async 가 선언되어 있어야 합니다.

     예)
     router.post('/login', async (req, res) => {   <-- async 필수
       ...
       await sendTelegram('메시지');
       ...
     });

──────────────────────────────────────────────────────────────
STEP 1 · 2 — 봇 생성과 CHAT_ID 확인을 왜 둘 다 해야 하나?
──────────────────────────────────────────────────────────────

텔레그램 알림은 "서버 → 텔레그램 서버 → 나의 채팅방" 순서로 전달됩니다.

이 과정에서 두 가지가 반드시 필요합니다.

  ① BOT_TOKEN : "누가 보내는가"를 텔레그램 서버가 식별하는 열쇠입니다.
                 없으면 텔레그램은 요청 자체를 거부합니다. (401 Unauthorized)

  ② CHAT_ID   : "어디로 보내는가"를 지정하는 주소입니다.
                 전화번호 없이 문자 못 보내는 것과 같습니다.

봇을 먼저 만들어야 TOKEN을 얻을 수 있고,
내 채팅방 ID를 알아야 메시지가 나에게 도착합니다.
이 둘은 한 쌍이기 때문에 순서대로 모두 완료해야 합니다.

또한, 새로 만든 봇에게 반드시 /start 를 먼저 보내야 합니다.
텔레그램 정책상 봇은 먼저 대화를 시작한 사용자에게만 메시지를 보낼 수 있습니다.
/start 를 보내지 않으면 서버가 아무리 알림을 보내도 수신이 차단됩니다.


──────────────────────────────────────────────────────────────
STEP 3 — .env 파일에 넣는 이유
──────────────────────────────────────────────────────────────

토큰과 ID 값을 코드 안에 직접 쓰는 것을 "하드코딩"이라고 합니다.

  // ❌ 나쁜 예 — 절대 이렇게 하지 마세요
  const BOT_TOKEN = '123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ';

이렇게 하면 GitHub에 코드를 올리는 순간 전 세계에 토큰이 공개됩니다.
악의적인 사람이 이 토큰을 가져가면 당신의 봇을 마음대로 사용할 수 있습니다.

.env 파일은 코드와 비밀값을 분리하기 위한 파일입니다.
process.env.TELEGRAM_BOT_TOKEN 처럼 환경변수로 불러오면
코드에는 값이 없고 실행 환경에서만 값이 주입됩니다.

.gitignore에 .env 를 추가하는 이유도 같습니다.
.gitignore는 GitHub에 올리지 않을 파일 목록입니다.
.env 가 여기 없으면 git add . 한 번에 토큰이 올라갑니다.


──────────────────────────────────────────────────────────────
STEP 4 — node-fetch 버전 2를 설치해야 하는 이유
──────────────────────────────────────────────────────────────

node-fetch 는 버전 3부터 ESM(import/export) 방식만 지원합니다.
HJS 프로젝트는 CommonJS(require/module.exports) 방식을 사용합니다.

만약 버전 3을 설치하면 아래와 같은 에러가 납니다.

  Error [ERR_REQUIRE_ESM]: require() of ES Module not supported

이것은 Node.js에서 두 모듈 시스템이 서로 호환되지 않아서 발생합니다.
node-fetch@2 는 CommonJS를 지원하는 마지막 버전이기 때문에
버전을 명시해서 설치해야 합니다.

  npm install node-fetch@2   <-- 반드시 @2 붙이기


──────────────────────────────────────────────────────────────
STEP 5 — telegram.js 를 별도 파일로 분리하는 이유
──────────────────────────────────────────────────────────────

텔레그램 발송 코드를 각 라우터마다 중복해서 쓸 수도 있습니다.
하지만 그렇게 하면 나중에 문제가 생깁니다.

  ① 텔레그램 API 주소가 바뀌면? → 모든 파일을 다 열어서 수정해야 함
  ② 에러 처리 방식을 바꾸고 싶으면? → 마찬가지로 전부 수정해야 함
  ③ 발송 실패 시 로그를 남기고 싶으면? → 또 전부 수정

config/telegram.js 한 곳에 모아두면
수정이 필요할 때 이 파일 하나만 고치면 됩니다.
이것을 "단일 책임 원칙" 또는 "코드 재사용"이라고 부릅니다.
실무에서 가장 기본이 되는 코드 관리 방식입니다.

또한 try-catch 로 에러를 감싸는 이유는,
텔레그램 전송이 실패해도 서버 자체가 멈추면 안 되기 때문입니다.
알림 실패는 부가 기능의 실패이지, 서버 전체의 문제가 아닙니다.
try-catch 없이 에러가 나면 해당 요청 전체가 500 에러로 끝납니다.


──────────────────────────────────────────────────────────────
STEP 6 — await 를 꼭 써야 하는 이유
──────────────────────────────────────────────────────────────

sendTelegram 함수는 async function 입니다.
async 함수는 호출하면 즉시 Promise 객체를 반환합니다.
await 없이 호출하면 Promise가 완료될 때까지 기다리지 않고
코드가 그냥 다음 줄로 넘어갑니다.

  // ❌ await 없이 호출하면
  sendTelegram('메시지');    // 완료 여부 모름, 에러도 못 잡음
  res.json({ ok: true });   // 이미 응답해버림

  // ✅ await 와 함께 호출하면
  await sendTelegram('메시지');   // 전송 완료 또는 실패 확인 후
  res.json({ ok: true });         // 그 다음에 응답

await 를 사용하려면 그 함수가 async 로 선언되어 있어야 합니다.
router.post('/경로', async (req, res) => { ... }) 에서
async 가 없으면 await 를 쓸 수 없습니다.

STEP 7 — 테스트를 마지막에 하는 이유
──────────────────────────────────────────────────────────────

개발에서는 "만들고 → 반드시 직접 확인" 이 원칙입니다.
코드가 아무리 잘 작성되어 있어도 환경변수 오타 하나,
봇에게 /start 를 안 보낸 것 하나로 알림이 오지 않습니다.

테스트 단계에서 확인해야 하는 이유:
  ① .env 값이 실제로 잘 로드되는지 확인
  ② 봇과 채팅방 연결이 정상인지 확인
  ③ 원하는 시점에 알림이 발생하는지 확인
  ④ 알림 메시지 내용이 읽기 좋은 형태인지 확인

터미널 로그와 텔레그램 수신 두 곳을 동시에 보면서 테스트하면
어디서 문제가 생겼는지 빠르게 파악할 수 있습니다.
